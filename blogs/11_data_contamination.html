Rethink BFCL - Data Contamination Detection

Fanjia Yan* Huanzhi Mao* Charlie Cheng-Jie Ji* Shishir G. Patil Tianjun Zhang

In the V2.0 release of the Berkeley Function-Calling Leaderboard (BFCL), we introduced a more advanced Abstract Syntax Tree (AST) evaluation checker. Previously, we broadly introduced the evaluation pipeline in the BFCL V1 blog. In this blog, we want to dive deeper and provide a comprehensive manual on how the scores on the leaderboard are calculated.

Evaluation Categories

Our system categorizes evaluations into four types:

  • Simple Function
  • Multiple/Parallel/Parallel-Multiple Function
  • Executable (Except REST API)
  • Executable (REST API Only)

Each category has a dedicated checker, ensuring nuanced and accurate assessment across different function call types. For all these fields, we always use the human-verified possible answers as the ground truth.
