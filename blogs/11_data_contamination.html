Rethink BFCL - Data Contamination Detection

Fanjia Yan* Huanzhi Mao* Charlie Cheng-Jie Ji* Shishir G. Patil Tianjun Zhang

Introduction

In the rapidly evolving field of Large Language Models (LLMs), the Berkeley Function-Calling Leaderboard (BFCL) emerges as a pioneering platform that evaluates the function-calling (tool-use) capabilities of LLMs such as GPT, Claude, Gemini, Llama, and Mistral. This initiative addresses the growing use of LLM for function-calling across various applications, highlighting the need for a comprehensive and fair assessment of their ability to execute functions (or tools) accurately in diverse contexts, including handling parallel and multiple function calls in various programming languages.

To overcome the limitations of traditional dataset-based evaluations, such as data contamination and the inability of static datasets to keep pace with rapid advancements in LLM capabilities and evolving use cases, we introduce BFCL-Live. Traditional evaluations often suffer from data contamination, where models are tested on the data they were trained on, leading to inflated performance metrics that do not accurately reflect their effectiveness.

BFCL-Live employs live, user-provided function documentation and queries, avoiding the drawbacks of dataset contamination and outdated benchmarks. By utilizing user-provided data, BFCL-Live aims to more faithfully measure the LLM's function-calling performance in real-world scenarios, emphasizing the importance of models performing effectively in diverse and dynamic environments.

BFCL-Live features a diverse set of 2,000 question-function-answer pairs, including novel function documentation like "multiple function with 10+ functions" and "complex function doc with 10+ parameters / nested parameters.” The dataset also includes fresh user queries that vary in tone, are part of multi-turn interactions, cover specialized use cases, and span multiple languages. This dataset will be assessed using the BFCL Abstract Syntax Tree (AST) evaluation pipeline to ensure thorough and accurate analysis.

Methodology

Data Pre-Processing

Data preprocessing is essential for generating structured and fresh user data. This process includes several key steps:

  1. Deduplication through ROGUE Score. To ensure uniqueness, we employ the ROGUE-L score in two distinct ways:

    • Query Deduplication

      Deduplicate user queries against existing datasets identified from NexusRaven, Firefunctions, Anyscale, and NousResearch-Glaive-Function-Callin.

    • Documentation Deduplication

      Repeated user-provided function documentation is deduplicated. This is crucial as some users may submit the same or slightly modified documents multiple times for testing.

  2. Query Deduplication through Text Embedding Model. We use the text-embedding-3-small model to further deduplicate user queries, enhancing the uniqueness of our data.
  3. We parse the user-provided function documents to determine the number of functions they contain and to categorize them as simple, multiple, parallel, or parallel-multiple.

Dataset Cleaning

Filtering of Low-Quality Data

Real-world user data often presents challenges due to its messy nature. This includes low-quality data that typically results from inadequately documented functions and prompts, which may lead to non-unique model responses and thus suboptimal for benchmarking purposes. In this section, we will describe how we define, determine, and manage low-quality data to enhance its utility in downstream applications. After transformation, these function documents will be compatible with our current BFCL evaluation procedure.

High-Quality Function Document
The function document should have a clear JSON structure that includes essential fields such as the function's name, function description, and detailed information on each parameter. This information includes parameter type, detailed parameter description (eg, its format and possible value range), and the default value if it's an optional parameter.
High-Quality User Query
The user query should align well with the function's intended use, utilize all required parameters effectively, and clearly specify all relevant parameters.
